# 贡献指南

感谢你关注 AIGC 影视制作大师 skill！本 skill 随行业快速演进，欢迎所有人贡献。

---

## 行为准则

- **诚实原则优先**: 不知道就说不知道，不编造工具名称、价格、论文标题
- **创意导演视角**: 贡献内容应服务于「判断力」和「审美策展」，而非单纯工具列表
- **中文优先**: 中文社区是核心用户，所有新增内容以中文为主
- **引用可查**: 事实性陈述应有可验证的来源（URL、人名、时间）

---

## 如何贡献

### 方式一：提交 Issue

适用于：
- 报告过时信息（工具更新、链接失效）
- 建议新增内容（新的类型片、新的行业趋势）
- 发现错误或歧义

Issue 类型：
- `🐛 Bug` — 信息错误
- `📝 更新` — 信息过时
- `💡 新增` — 建议补充内容
- `❓ 提问` — 确认信息

### 方式二：提交 Pull Request

适用于：
- 更新工具信息（新增/移除/定价变更）
- 新增类型片 Playbook
- 补充调研薄弱维度
- 修复错误

---

## PR 流程

1. **Fork** 本仓库
2. 创建分支：`git checkout -b feat/your-feature` 或 `fix/issue-number`
3. 在对应目录修改：
   - 工具更新 → `references/research/02-tools.md`
   - 行业动态 → `references/research/01-figures.md`
   - 工作流变更 → `references/research/03-workflows.md`
   - 类型片 → `references/genre-playbooks/*.md`
   - 核心框架 → `SKILL.md` 或 `references/synthesis.md`
4. 提交：`git commit -m "feat: add XXX"`（遵循 [Conventional Commits](#commit格式)）
5. Push：`git push origin feat/your-feature`
6. 打开 Pull Request，描述修改理由和来源

---

## Commit 格式

```
<type>: <简短描述>

[可选正文：解释为什么]

[可选 Footer：关联 Issue]
```

### Type

| Type | 用途 |
|------|------|
| `feat` | 新增内容（工具/类型片/模型）|
| `fix` | 修复错误信息 |
| `update` | 更新过时内容 |
| `refactor` | 重构（格式/结构优化）|
| `docs` | 文档更新 |
| `chore` | 维护性更改（依赖/配置）|

### 示例

```
feat: 新增可灵 3.0 基准数据

更新工具地图中可灵最新 Arena ELO 排名和生成时长限制。
来源：可灵官网 2026-05，VBench 2026-04 榜单。

Closes #12
```

```
fix: 修正 Runway Gen-4 价格

原文档价格已更新，修正为 $35/月（Standard 档）。
来源：Runway 官网定价页 2026-05-20。
```

## 内容标准

### 必须包含

- 明确的来源（URL/人名/时间）
- 可验证的具体信息（不是泛泛的形容词）
- 中文语境优先的表述

### 禁止包含

- 未验证的工具名称或功能描述
- 具体价格（注明来源和日期，或标注「需核实」）
- 版权敏感内容（如直接引用他人作品）
- 个人攻击或主观审美判断作为事实陈述

### 推荐格式

**工具描述**：
```markdown
### [工具名] [版本]

- **用途**: <一句话>
- **定价**: [具体价格]（来源: [URL]，[日期]）
- **核心能力**: [可量化描述，如「10秒 1080P」「支持唇形同步」]
- **已知局限**: [可量化描述，如「文字生成不稳定」「一致性约70%」]
- **适用场景**: [具体，不要泛泛]
```

**人物描述**：
```markdown
### [姓名]

- **身份**: [公司/职位]
- **核心贡献**: [具体，不要「推动了行业发展」这种泛泛表述]
- **公开观点**: [引用原文 + 来源 + 日期]
- **代表作品**: [如有]
```

## 评审流程

- 维护者会在 48h 内回复所有 Issue 和 PR
- 重大更改（新增章节、修改核心心智模型）需要至少 2 个 Approve
- 工具价格/版本更新可由单人快速合并
- 争议内容会进入 Discussion 线程，不会直接合并
