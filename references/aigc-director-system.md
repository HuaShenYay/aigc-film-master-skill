# AIGC 导演系统

> 在故事/剧本开发之后、进入 Image2/Seedance 提示词之前使用。

## 职责定义

AIGC 导演不是简单描述实拍剧组怎么拍，而是把**故事意图**翻译成**模型可生产的视觉和时序控制**。

---

## 导演通道

对每个场景或段落，定义：

- **戏剧功能（Dramatic Job）**：观众必须理解或感受什么
- **视角（POV）**：谁的认知/情感主导
- **AIGC 可行性**：直接 / 资产优先 / 故事板优先 / 后期优先 / 拆分 / 规避
- **视觉规则**：色调、镜头、光线、构图、重复物品或身体行为
- **表演规则**：呼吸、沉默、微表情、手势、台词节奏
- **段落策略**：单镜、双镜、三镜、蒙太奇、拆分动作、后期叠加
- **入状态（In State）**：角色、道具、场景、情绪在段落开始时
- **出状态（Out State）**：交给下一个段落的精确状态

---

## AIGC 可行性标签

| 标签 | 含义 | 动作 |
|------|------|------|
| `DIRECT_GENERATABLE` | 单动作、少角色、空间清晰 | 可进入下游 |
| `ASSET_REQUIRED` | 身份/道具/场景必须先锁定 | 先建资产板 |
| `BOARD_REQUIRED` | 镜头/blocking/动作必须画出来 | 先建故事板 |
| `POST_REQUIRED` | 精确文字/UI/VFX/音频更适合后期 | 加入后期职责 |
| `SPLIT_REQUIRED` | 内容超出 15 秒段落容量 | 拆分成更小段落 |
| `AVOID` | 不稳定或不值得直接生成 | 重新设计节拍 |

## 导演护栏

1. 导演想法在变成资产、故事板、Seedance、后期、拆分或规避职责之前，不算完成
2. **不要**让 Seedance 生成精确的中文段落、字幕、手机 UI、合同、倒计时
3. 不要在一个段落里放超过一个主要情绪转折
4. 慢场景可以很强；不要对情绪节拍过度剪辑
5. 终画必须尽量回溯开场画面或母题
