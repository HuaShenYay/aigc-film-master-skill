# 短剧 Playbook

> 适用场景：竖屏短剧、AI 真人短剧、漫剧单集、连续投流短剧、强冲突剧情广告。

## 观众承诺

观众要**快速清晰、强冲突、情感释放**，以及继续看下一节拍或下一集的理由。

---

## 剧集引擎

| 元素 | 要求 |
|------|------|
| 冷开场 | 几秒内出现冲突、震惊、疑问或屈辱 |
| 欲望 | 主人公想要某个可见的东西 |
| 压力 | 反派/系统立刻阻挡 |
| 反转 | 结束前权力/信息发生变化 |
| 情感钩子 | 观众选边站 |
| 悬念结尾 | 下一个动作变得不可避免 |

---

## 场景设计

- 每集优先 **2-4 个可控制场景**
- 每段**有名角色数量保持低**。两人对峙比群戏更稳定
- 用道具作为权力物品：手机、合同、戒指、包、药品、钥匙、文件
- **对白必须精确可表演**；长台词按段落拆分
- **第一帧**必须立即显示关系压力或后果

---

## 镜头与调度（9:16 竖屏）

- 9:16 画幅：**脸、手、手机、文件、门、桌子**成为重要垂直锚点
- 使用：过肩镜头、特写、插入镜头、反应停顿
- **避免**：除非是背景纹理，否则避免广阔人群场景
- **确保**：视线和权力距离清晰
- **悬念结尾画面**：必须简单到可以作为缩略图

---

## 节奏

| 时长 | 结构 |
|------|------|
| 0-3s | Hook |
| 3-15s | 冲突规则 |
| 15-45s | 升级和第一次反转 |
| 45-90s | 情感压力或秘密 |
| 最后 3-8s | 悬念结尾、揭示、威胁、选择、反击台词 |

---

## 声音

- 对白承载冲突，但台词后的**沉默**往往更能卖出反转
- 使用：手机震动、消息提示音、关门声、脚步、文件拍桌、呼吸
- **字幕是后期**，不是 Seedance 帧内生成的

---

## AIGC 风险与修复

| 风险 | 修复方式 |
|------|----------|
| 角色太多 | 拆分成更小的对峙段落 |
| 对白太长 | 精确碎片化续接 |
| 手机/文件文字损坏 | UI 文字板或后期 |
| 跨集脸漂移 | 角色板 + 状态表 |
| 表演过激 | 微表情和反应计时 |

---

## 下游提示词策略

- **角色板**：所有 recurring 角色
- **道具/UI 板**：手机、合同、病历、聊天屏幕
- **故事板**：聚焦对峙 blocking 和反应顺序
- **Seedance 提示词**：包含精确对白碎片作为 audio/performance，不生成字幕
- **用前集结尾帧**：仅作为开场状态，不覆盖角色身份

---

## 质感锁定

```text
短剧质感：开场即有关系压力；每段只承载一个冲突动作或信息反转；角色脸和服装连续，台词按声音表演处理，不生成字幕；结尾画面必须形成下一段钩子。
```

```text
Short-drama look: immediate relationship pressure, one conflict action or reveal per segment, locked faces and wardrobe, dialogue as spoken performance not rendered subtitles, final frame creates the next hook.
```
