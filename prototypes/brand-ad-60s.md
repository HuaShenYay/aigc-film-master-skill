# 原型: 60 秒 AIGC 品牌广告制作全流程

> **场景**: 一个新消费品牌（假设为「茶颜悦色」）需要一支 60 秒的品牌形象广告，用于抖音+B站投放。
> **约束**: 预算 ¥5000 以内，7 天交付，需要品牌 IP 形象（古风少女）跨镜头一致。
> **目标**: 展示从 0 到成片的完整 AIGC 工作流，体现本 skill 的所有核心决策点。

---

| 字段 | 内容 |
|------|------|
| 项目名称 | 「茶香千年」品牌形象广告 |
| 时长 | 60 秒 |
| 画幅 | 9:16（竖屏）+ 16:9（横屏备份） |
| 目标平台 | 抖音（主）、B站（备份） |
| 预算 | ¥5000（工具订阅 ¥800 + 配音 ¥200 + 备用 ¥4000） |
| 交付周期 | 7 天 |
| 核心挑战 | 古风少女 IP 形象跨 8 个镜头保持一致 |

## 决策过程（体现心智模型）

### 决策 1: 工具选型 → 触发 M5（亚洲审美溢价）

**问题**: 用什么视频生成工具？

**分析**:
- 目标观众：中国，古风审美，亚洲人脸
- 竞品参考：蜜雪冰城 AI 广告用了可灵
- 结论：**可灵 Kling 2.0**（主）+ **即梦 AI**（辅助分镜）+ **Runway Gen-4**（备选，用于特殊运镜）

**选型决策树**:
```
预算 ¥5000 → 可灵专业版 ¥299/月 × 2 个月 = ¥598 ✅
目标中国市场 → 可灵 > Runway（M5 亚洲审美溢价）✅
需要角色一致 → 可灵 + LoRA 训练 ✅
```

---

### 决策 2: 一致性方案 → 触发 M2（一致性是第一公民）

**问题**: 8 个镜头里古风少女怎么保持同一张脸？

**方案选型**:
| 方案 | 成本 | 效果 | 选择 |
|------|------|------|------|
| seed 锁定 | ¥0 | 弱（同一 prompt 不同 seed 仍会脸变） | ✗ |
| IP-Adapter | ¥0，ComfyUI 技术门槛高 | 强 | △（团队不会 ComfyUI） |
| **LoRA 训练** | ¥0（自己训练），耗时 2 小时 | **最强** | ✅ |

**执行**:
1. 用 Midjourney 生成 15 张「古风少女 IP 参考图」（固定 prompt，变 seed）
2. 用可灵内置 LoRA 训练功能上传 15 张图，训练 25 步
3. 训练完成后，所有生成调用这个 LoRA，权重 0.7

---

### 决策 3: 生成策略 → 触发 M3（抽卡经济学）

**问题**: 每个镜头要生成多少次？

**出片率预估**（基于行业数据）:
- 可灵 + LoRA 锁定：出片率 ~50%（比无 LoRA 的 30% 高）
- 8 个镜头 × 每个镜头生成 10 条 = 80 条素材
- 预期可用：80 × 50% = **40 条**
- 实际需要：8 个镜头 × 每个镜头 1 条 = **8 条**
- 安全边际：40 条 >> 8 条 ✅

**Batch 策略**:
```
每个镜头：
  Loop 10 次：
    生成 1 条（可灵 API 批处理，同时跑 3 个）
  人工挑选 1-2 条最好的
  如果 10 条都不行 → 调整 prompt → 再生成 5 条
```

## 完整时间线（7 天）

### Day 1-2: 前期（占总工时 40%）

```
Day 1 上午（3h）:
  - ChatGPT 辅助写 60 秒脚本（8 个镜头，每个镜头 6-8 秒）
  - 确定叙事结构：茶文化历史 → 现代年轻人手里的一杯茶 → 品牌连接古今

Day 1 下午（4h）:
  - Midjourney 生成古风少女 IP 参考图（15 张）
  - 可灵 LoRA 训练（上传 15 张，训练 25 步，耗时 2h）

Day 2 全天（8h）:
  - Midjourney 生成每个镜头的 keyframe（8 个镜头 × 3 张变体 = 24 张）
  - 写每个镜头的提示词（结构化：主体+动作+背景+运镜+光影+色调）
  - 即梦 AI 生成分镜板（12 格，用于客户确认）
```

### Day 3-4: 生成（占总工时 30%）

```
Day 3（8h）:
  - 可灵批量生成：镜头 1-4，每个镜头生成 10 条变体
  - 人工挑选：每个镜头标记 1-2 条可用素材
  - 失败处理：镜头 2 出片率只有 20%（prompt 问题），重写 prompt，重新生成 10 条

Day 4（6h）:
  - 可灵批量生成：镜头 5-8
  - 人工挑选
  - 一致性检查：把 8 个镜头的精选素材并列，确认古风少女脸一致
  - 不一致处理：镜头 6 的脸偏年轻，重新生成（调整 LoRA 权重 0.7→0.8）
```

### Day 5-6: 后期（占总工时 25%）

```
Day 5（8h）:
  - 剪映 Pro 导入 8 个镜头精选素材
  - 粗剪：按脚本排列，调整时长（每个镜头精确到帧）
  - 调色：用剪映「滤镜」+ 手动调整，确保所有镜头色调一致（避免色调漂移）
  - 加转场：用 AI 生成转场（剪映 Pro 内置）

Day 6（6h）:
  - 配音：Elevenlabs 中文配音（古风少女旁白，「温柔女声」模型）
  - 配乐：Suno 生成古风电子混音（prompt: "Chinese ancient style, modern electronic, 60s, brand anthem"）
  - 音效：剪映音效库（倒茶声、风声、城市环境音）
  - 口型同步：Runway Gen-4 的 lip-sync 功能（镜头 3 特写需要口型对齐）
```

### Day 7: 交付（占总工时 5%）

```
Day 7（4h）:
  - 质量检查：逐帧检查手部畸变、色调漂移、音频同步
  - 多版本导出：9:16 竖屏（抖音）+ 16:9 横屏（B站备份）
  - AI 标注：片尾加「本片含 AI 生成内容」字幕（中文+英文，持续 3 秒）
  - 客户交付：输出 H.264 MP4，码率 15Mbps，音频 AAC 320kbps
```

## 质量自检（用 2.6 基准验证）

| 基准 | 目标 | 实际 | 达标？ |
|------|------|------|--------|
| 跨镜头角色一致性 | ≥70% 观众认出同一角色 | 8 个镜头脸一致，LoRA 权重 0.8 | ✅ |
| 出片率 | ≥40% | 实际 45%（36/80） | ✅ |
| 音频-画面同步误差 | <200ms | Elevenlabs + 剪映对齐，误差 <100ms | ✅ |
| 叙事密度 | ≥1 个情绪转折点/30s | 60 秒有 3 个转折（历史→现代→品牌） | ✅ |
| AI 标注合规 | 完播率下降 <5% | 片尾 3 秒标注，预计下降 2-3% | ✅ 预计 |

## 反模式检查（避坑）

| 反模式 | 有没有犯 | 处理方式 |
|--------|----------|----------|
| AI 味运镜 | ❌ 没有 | 每个镜头的运镜都在分镜阶段设计意图，不是随机平移 |
| 手部畸变 | ⚠️ 镜头 5 有轻微手指粘连 | Day 6 后期用 DaVinci 修复（roto + 手动绘景） |
| 色调漂移 | ❌ 没有 | Day 5 统一调色，所有镜头用同一个 LUT |
| 音频缺失 | ❌ 没有 | 配音+配乐+音效三层齐全 |
| 叙事断裂 | ❌ 没有 | 脚本经过 ChatGPT 辅助结构审查 |
| 文字错误 | ⚠️ 镜头 4 有招牌，文字扭曲 | 用 PS 手动修复招牌文字（10 分钟） |
| 接缝闪烁 | ❌ 没有 | 剪映转场 + 统一调色避免跳变 |
| 版权合规 | ❌ 没有 | AI 标注已加，古风少女是原创 IP 不侵权 |

## 复盘：如果重来，我会…

1. **更早做 LoRA** — Day 1 就训练好，而不是 Day 2 结束才完成。这样 Day 3 生成时更有信心
2. **预留更多生成预算** — 镜头 2 的 prompt 重写花了半天，应该在前期多花时间打磨提示词
3. **音频先定** — 这次是画面先定再配乐，导致一段 8 秒的镜头配乐情绪不对，重生成了配乐
4. **避免使用文字** — 镜头 4 的招牌文字是个坑，下次类似场景用「无文字」版本生成再后期加文字

## 给用户的启示

看完这个原型，你应该理解：

1. **AIGC 影视不是「写一句 prompt 就出片」** — 7 天工时有 40% 花在前期（脚本+分镜+LoRA），只有 30% 花在生成
2. **一致性是有成本的** — LoRA 训练 2 小时 + 调试权重，但这是专业和业余的分水岭
3. **出片率是可以规划的** — 知道出片率 50%，就规划生成 2 倍于需要的素材量
4. **音频占工时的 25%** — 忽略音频的「AI 导演」做出来的东西一看就是「AI 味」
5. **¥5000 可以做出专业级 60 秒广告** — 传统方式可能需要 ¥50000+，这是 AIGC 的真实价值
